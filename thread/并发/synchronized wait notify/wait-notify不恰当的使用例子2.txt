// wait场景
while(!condition) {
	synchronized(obj) {
　　		obj.wait();
　　}
　　obj.doSomething();
}

// notify场景
synchronized(obj) {
	obj.doPrepare();
　	condition = true;
	obj.notify();
}

问题：
wait和notify的逻辑 在不同的线程上执行，当执行完wait的while中的条件判断后（设condition为false），开始执行notify的逻辑，并执行完毕，后继续wait中的剩余逻辑
随后，obj.wait被执行，但是之后，再没有线程执行obj.notify，则wait逻辑永远不会再被唤醒
